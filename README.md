Система подання роботи на конкурс (Варіант 21)
Цей проєкт демонструє реалізацію сценарію подання конкурсної роботи з використанням принципів Гексагональної архітектури (Ports and Adapters). Головна мета — відокремити стабільну бізнес-логіку від мінливих технічних деталей.

📝 Опис Use Case
Назва: Подання конкурсної роботи.
Сценарій:

Користувач вводить назву роботи, автора та обирає категорію.

Система перевіряє, чи входить обрана категорія до списку дозволених.

Система перевіряє, чи не минув дедлайн для подання робіт.

Якщо всі правила дотримані — робота реєструється та зберігається у сховищі.

🏛 Обрана архітектура: Hexagonal Architecture
У цьому проєкті використано підхід Ports and Adapters.

Обґрунтування вибору:
Незалежність від інфраструктури: Бізнес-логіка не знає, куди зберігаються дані (у базу даних, файл чи просто в пам'ять).

Легке тестування: Ми можемо протестувати процес подання роботи, не підключаючи реальну базу даних.

Чіткі межі: Кожен шар має свою зону відповідальності.

📂 Структура проєкту та розподіл відповідальностей
Проєкт розділений на логічні шари:

src/domain (Core / Entities):

Містить опис сутності CompetitionWork.

Тут знаходяться "чисті" бізнес-правила (наприклад, список валідних категорій).

Цей шар не має жодних зовнішніх залежностей.

src/application (Use Cases / Ports):

Use Case (SubmitWorkUseCase): Координує процес. Саме тут реалізована перевірка дедлайну та логіка "якщо все ок — збережи".

Ports (Interfaces): Описує інтерфейс IWorkRepository. Це "розетка", в яку має ввімкнутися будь-яке сховище.

src/infrastructure (Adapters):

Містить технічну реалізацію — InMemoryRepository.

Це "вилка", яка реалізує інтерфейс порту і зберігає дані в масив (імітація бази даних).

src/main.ts (Entry Point):

Точка збірки додатка. Тут ми створюємо екземпляри адаптерів, "впорскуємо" їх у Use Case (Dependency Injection) та запускаємо сценарій.

✅ Реалізовані бізнес-правила
У коді реалізовано наступні перевірки:

Валідація категорії: Робота приймається лише в категоріях: Art, Science, IT.

Перевірка дедлайну: Система порівнює поточний час із встановленим дедлайном конкурсу. Якщо час вичерпано — повертається помилка.

🚀 Як запустити проєкт
Оскільки проєкт написаний на TypeScript:

Встановити залежності: npm install (якщо використовуються зовнішні пакети).

Запустити через: npx ts-node src/main.ts.

Результати виконання (успішне подання або помилки валідації) будуть виведені в консоль.
